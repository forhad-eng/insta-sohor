JavaScript is a high-level, single-threaded, asynchronous, non-blocking, event-driven, prototype-based object oriented programming language. In case of Chrome, JavaScript run on V8 engine. At first, V8 engine create an Abstract Syntax Tree of JS source code and then run it through the Interpreter and JIT compiler pipelines. Interpreter follows a data structure called stack. In stack, each function call is placed on top as a stack frame. Where asynchronous function send back to the browser API. Having done job by the browser API, then send callback function to the queue and afterwards Event Loop push them to the call stack subject to the empty call stack.

In the Interpreter when an asynchronous function comes in the call stack, for instance setTimeout(), fetch() etc. Then these function send back to the corresponding browser API to do their job. When browser API completes the task afterwards it sends the callback function the the queue. Here Event Loop comes in. The task of Event Loop is to push the callback function into call stack from the queue. Before doing this Event Loop checks whether the call stack is empty. If so then Event Loop push the callback function to the call stack.

Local Storage: Local Storage is a long-term storage of data for the individual tab inside the browser. It keeps data in JSON format. To manipulate the Local Storage, we have several methods on hand. getItem() is used to fetch the data from the storage and setItem() is used to store the data. Where removeItem() is used to remove the key from the storage.

Session Storage: Session Storage is a temporary storage of data for the individual tab inside the browser. Data will be lost if we close the tab or close the browser. It also keeps the data in JSON format and it's functional work is as same as the Local Storage.
